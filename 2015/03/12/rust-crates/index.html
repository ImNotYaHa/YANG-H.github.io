Rust - 箱子（Crates）

Mar 12 2015
  • 技术
  • Rust
  • 学习 Rust

原文在此

一个箱子（crate）代表着 Rust 中的一个编译单元。只要执行rustc some_file.rs，some_file.rs就会被视作一个箱子文件（crate file）。如果some_file.rs中有声明任何mod，那么相关的模块成员都会在编译器编译之前合并到这个箱子文件中去。也就是说，模块不是被单独编译的，只有箱子文件可以被编译。

一个箱子既可以被编译成可执行程序，也可以被编译成库。rustc默认将箱子文件编译成可执行程序。你可以通过在rustc后添加-crate-type标记来更改这一行为。

10.1 库

这里我们先创建一个库，后面将尝试把它链接到另外一个箱子上。

生成的库文件默认使用其对应的箱子文件的名字，同时自动添加一个”lib”前缀。这一行为可以通过使用 crate_name 属性来加以更改。

10.2 extern crate

若想让一个箱子链接一个新的库，你需要使用extern crate声明。这不只能够起到链接某个库的作用，这还会通过一个与此库同名的模块来导出这个库的成员。这些库成员的可见性规则与模块相同。
